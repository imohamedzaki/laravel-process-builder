**Bugs found and fixed while building this milestone:**

1. `ProcessRollbackController` originally reconstructed each managed file's current absolute path via `base_path($entry->relativePath)`, which silently breaks for any process whose `process-builder.output.*` config points outside `base_path()` (a config option the package itself exposes). Fixed by having `ManifestEntry` persist the exact `absolutePath` used at generation time (the manifest is trusted package-internal storage populated from the compiler's own output, not browser input, so this doesn't reopen the "no arbitrary paths from the client" rule) and having the rollback controller read it directly instead of re-deriving it.
2. `BackupService`'s relative-path-to-backup-filename encoding (`str_replace(['/', '\\'], '__', $relativePath)`) didn't escape `:`. When a `relativePath` ends up as a Windows absolute path with a drive letter (e.g. because output config points outside `base_path()`, triggering the fallback in `ProcessCompiler::toProjectRelativePath()`), the un-escaped colon after the drive letter truncated the backup filename (`C:` is a reserved Windows path prefix), silently corrupting the backup and causing `RollbackFailedException: source or destination path unknown`. Fixed by escaping `:` alongside `/` and `\` in `BackupService::encodeRelativePath()`, with a regression test covering a Windows-drive-letter-style relative path.
